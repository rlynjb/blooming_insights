# `.aipe/study-distributed-systems/`

Per-codebase study guide on **distributed systems** for **blooming_insights**. Generated by `/aipe:study-distributed-systems`.

The honest framing up front: this repo has **one distributed wire surface** — HTTPS to the Bloomreach loomi-MCP server. Everything else (the in-process synthetic data source, the per-instance caches, the agent loop) is single-process. Most of distributed-systems theory shows up here as **Case B — not exercised**, with a short explanation of when it would start mattering.

## What the file list tells you

The two Case B files (`05`, `08`) and the partial Case B in `07` are deliberate. A guide that pretended this repo did quorum reads or saga orchestration would teach the wrong shape. The audit (`09`) ranks the real risks — not the theoretical ones.
