# 01 — Reasoning patterns

Anchor: single-agent (primary) · workflow (secondary)

How one model thinks through a task. Every agent in this repo is one instance of these patterns. Files are mostly self-contained; the recommended order is the file numbering.

## Reading order

1. `01-chains-vs-agents.md` — the boundary. Workflow outside, agent inside.
2. `02-agent-loop-skeleton.md` — the kernel every named pattern instantiates. Read this before any of the named patterns.
3. `03-react.md` — the default. Every agent in this repo is one of these.
4. `04-plan-and-execute.md` — not yet in this repo. When it would earn its place.
5. `05-reflexion-self-critique.md` — not yet in this repo. The recovery prompt is *not* this.
6. `06-tree-of-thoughts.md` — covered for completeness. Rarely worth it in production.
7. `07-routing.md` — `classifyIntent` is the live instance. The bridge to Section C.
