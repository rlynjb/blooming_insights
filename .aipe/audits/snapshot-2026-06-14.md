## Section 5 — Decisions

All 14 decisions from 06-02 still apply. Three updates plus one new decision recorded by code change:

**Updated — session-keyed in-memory state (was: deferred).** The 06-02 snapshot listed the `lib/state/insights.ts:4` global-Map issue under "Risks and rough edges." Commit `c3d47a2` made the call to fix it, with a 6-line code comment recording the reasoning (`lib/state/insights.ts:4-7`): "A single warm Vercel instance serves many users concurrently, so module-level Maps would bleed between sessions — and putInsights' clear() would wipe another user's feed mid-briefing." The fix preserves the per-session `.clear()` semantics (each briefing IS the current feed for that session) while closing the cross-session bleed.

**Updated — 30s per-tool MCP timeout (was: deferred).** Commit `9597466` added `AbortSignal.timeout(TOOL_TIMEOUT_MS)` to both `SdkTransport.callTool` and `listTools`. The decision and tradeoff are documented at `lib/mcp/transport.ts:21-30`: "Sibling of `retryCeilingMs: 20_000` in client.ts — that ceiling bounds a rate-limit retry wait, this one bounds the request itself." The retry ladder in `McpClient.callTool` only retries rate-limited *results*, not thrown errors, so a 30s timeout fails fast rather than burning another 30s wait inside the same route budget.

**Updated — tool-name allowlist on `POST /api/mcp/call` (was: deferred).** Commit `39721e2` added a `Set<string>` built from `monitoringTools ∪ diagnosticTools ∪ recommendationTools ∪ bootstrapTools` (`app/api/mcp/call/route.ts:14-19`). Decision recorded in code comment at `app/api/mcp/call/route.ts:11-13`: "Sourced from the same constants the agents use, so the boundary stays in sync with what the rest of the system already speaks." The /debug page was trimmed in the same commit to remove `whoami` (the test at `test/api/mcp-call-allowlist.test.ts:52-57` pins this end-to-end contract).

**New — server-side phase timing is the right grain for the 300s budget.** Commit `917d521`. The observability addition is `console.log({ route, sessionId, totalMs, phases })` in a `finally` block, NOT a new event type on the NDJSON wire (`app/api/agent/route.ts:222-228, 307-318`). Decision recorded in the comment: "the summary still fires when a phase throws (the 300s-budget incident signal). Not on the NDJSON wire. Shape matches /api/briefing so a single Vercel filter reads both routes." The cost meter (`res.usage` at the four Anthropic call sites — commit `cc43e7d`) and the secret-redaction helper (commit `56e405b`) are recorded in the same observability spirit: cheap logs that close the prior blind spots without adding a metrics pipeline.

**Updated — error-response bodies carry message-only, not stack.** Commit `410e2bb`. Bodies in `/api/mcp/call`, `/api/mcp/tools`, `/api/mcp/tools/check`, `/api/mcp/capture` now return `{ error: e.message }`. `console.error(e)` still fires so operators see the full stack in Vercel logs. Brings the four MCP routes into line with the streaming routes (`/api/briefing`, `/api/agent`), which were already on the safer shape.
