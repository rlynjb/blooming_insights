### The queue head — the first move routed to `/aipe:drill`

**Move 2 — shared-state Map race (`induce-shared-state-leak-session-key`, L1 → L2).**

- **Slug:** `l1-correctness-induce-shared-state-leak-session-key`
- **Competency:** L1 correctness — the 4×-named-and-unshipped fix
- **Induce:** Two `/api/briefing` requests with the same `sessionId` cookie hitting the same warm Vercel instance in flight simultaneously. First writer's `insights` array gets clobbered by the second writer's `putInsights()` at `lib/state/insights.ts:53–66` (the whole `SessionFeed` object is replaced, not merged). One user's briefing evaporates. Silent — no error thrown.
- **Diagnose:** `lib/state/insights.ts:4` — the shared `Map<sessionId, SessionFeed>`. `putInsights` at line 53 replaces the entire feed object per key. No mutex, no compare-and-set, no last-write-wins with a merge. Fresh cross-references: `.aipe/study-runtime-systems/04-shared-state-races-and-synchronization.md` walks the exact race; `.aipe/study-database-systems/05-transactions-isolation-and-anomalies.md` names it as the anomaly site; `.aipe/study-distributed-systems/09-distributed-systems-red-flags-audit.md` R2 has it as the shipped-code race.
- **Fix:** Two options (option matrix in the drill scaffold). (A) Session-keyed sub-map: `feeds.set(sessionId, [...oldFeed, ...newInsights])` with a shallow-merge helper — 5 LOC + 3 tests. (B) Replace the top-level Map with per-sessionId in-memory queues + a single-writer loop — cleaner concurrent semantics, but adds a coordinator layer (~40 LOC). Ship A; keep B as follow-up.
- **Prove with an eval:** Race-only test — spawn N concurrent `putInsights` calls for the same sessionId, assert all N writes land (in either order). Baseline: fails (last-write-wins wipes N-1). Fixed: passes.
- **Why this is second:** it's small, well-scoped, and named 4× — the receipt earns interview trust ("I found it, I traced it across four different design lenses, I shipped it in 30 LOC + tests"). And it's on the L1 dimension (correctness), which is where interviewers probe first before touching L3.
