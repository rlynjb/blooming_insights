# Recon — 2026-06-02

> **Verdict-first.** blooming insights is a strong **L1 with one L2 spike** LLM-app codebase. The build is real and ambitious for one engineer — a 4-agent sequential pipeline on a shared `runAgentLoop` (`lib/agents/base.ts:48–176`), a hand-built MCP client with TTL + spacing + bounded exp-backoff retry (`lib/mcp/client.ts:79–172`), NDJSON streaming with replay (`lib/mcp/events.ts:4–17`, `lib/state/investigations.ts`), and an encrypted-cookie OAuth boundary (`lib/mcp/auth.ts:51–79`) — but the recon rule is harsh on it: **no eval and no handled failure caps the ceiling at L1.** Zero LLM evals (no `evals/` directory, no judge, no goldset, no agreement rate — confirmed in `study-testing/audit.md` and `study-ai-engineering/05-evals-and-observability/`), four `anthropic.messages.create` call sites with `res.usage` dropped on the floor (`lib/agents/base.ts:102`, `lib/agents/diagnostic.ts:97`, `lib/agents/recommendation.ts:96`, `lib/agents/intent.ts:18`), and no circuit breaker on the MCP retry ladder (`lib/mcp/client.ts:121–132`) — three load-bearing absences in the failure-and-evidence layer the bar measures hardest. The L2 spike: ONE post-mortem with a real symptom→cause→fix arc (commit `e83a8e0`, the AUTH_SECRET parallel-worker flake fix, `test/mcp/auth.test.ts:117–122`). The load-bearing gap is **evals**; the load-bearing strength is **agents + tool-use + MCP orchestration**; the load-bearing liability is the resume-vs-reality flip on "production-grade AI" given the unmeasured cost and missing eval.

## 1. AUDIT — the AI/ML surface

### Shape: LLM-app (one of the three shapes)

This is unambiguously an **LLM-app** in the three-shapes framing — not classical-ML (no trained models, no offline data pipeline; the only "prediction" is `get_customer_prediction_score`, which is a Bloomreach-provided MCP tool, not a local model — `study-ai-engineering/00-overview.md:57`), and not embedding-RAG (deliberate: the corpus is the live Bloomreach API, retrieval is agentic EQL through MCP tools, no vector store anywhere — `study-ai-engineering/03-retrieval-and-rag/11-rag.md`). Every AI feature is "prompt + tools + loop + structured-output parse," wrapped in route handlers.

### Real vs scaffolded — what's hand-built, what isn't

**Real (built and understood):**
- `runAgentLoop` — the single ReAct kernel reused across all 4 agents, including the `forceFinal` tool-less turn pattern at `lib/agents/base.ts:88–101`. Not from a framework; written by hand.
- `parseAgentJson` + type-guard validators (`lib/mcp/validate.ts:1–57`) — the structured-output boundary. Fenced → bare → substring scan fallback chain.
- `McpClient` (`lib/mcp/client.ts:79–172`) — TTL cache, inter-call spacing (`minIntervalMs: 1100` for the ~1 req/s Bloomreach limit), exponential backoff with parsed Retry-After honoring (`:121–132`). Written, not borrowed.
- `synthesize()` recovery path (`lib/agents/diagnostic.ts:87–126`, `lib/agents/recommendation.ts:82–132`) — the tool-less forced-JSON fallback when the loop's final turn fails to parse.
- Capability gating (`lib/agents/categories.ts:1–160`) — schema-shape inspection before the monitoring agent runs, so the model never spends budget on categories the data can't support. This is rare and good — most L1 codebases skip it.
- Intent router (`lib/agents/intent.ts:1–31`) — heuristic + cheap-model classifier in front of the QueryAgent. Standard pattern, correctly applied.
- NDJSON streaming + replay (`lib/mcp/events.ts:4–17`, `lib/state/investigations.ts:11–41`) — discriminated-union `AgentEvent`s on a `ReadableStream`, with verbatim replay from cache. The trace is the product surface (`study-ai-engineering/05-evals-and-observability/README.md:5`).
- AES-256-GCM encrypted cookie store for OAuth (`lib/mcp/auth.ts:51–79`) — replaces a database for the session boundary.

**Scaffolded but unstressed (the L0 zone — interview liabilities):**
- The `model: AGENT_MODEL` choice (`lib/agents/base.ts:9`) — one line, sonnet-4-6 used uniformly for the 4 main agents, haiku for intent only. No decision artifact about why sonnet vs opus, no cost/quality tradeoff measured. A reviewer asks "why this model?" and the honest answer is "default."
- Prompt files themselves (`lib/agents/*.ts` system prompts) — written, edited, but not versioned, not A/B tested, not eval-scored. A prompt edit lands with zero quality signal.
- Sampling parameters (temperature, top_p) — defaulted. Not tuned. Not justified.

### The three usually-missing things — checked explicitly

**EVALS — absent.** No `evals/` directory at repo root (confirmed by `find`). No goldset. No judge model. No agreement-rate calibration. No regression set. The eval substrate (the NDJSON trace + `durationMs` + `lib/state/investigations.ts` replay) is built — replaying with a changed prompt against the live model *would be* the harness — but the harness itself is not written. `study-testing/audit.md` names this as **Case B**, top-3 finding #3: "169 green tests, zero quality signal." `study-ai-engineering/05-evals-and-observability/README.md:7` is just as blunt: "every prompt edit and model swap currently ships with zero quality measurement." **This is the load-bearing gap and it caps the LENS verdict at L1 by the recon rule.**

**FAILURE HANDLING — partial.** Eight handled paths named in `study-system-design/audit.md` (Bloomreach 429 retry, transport throw, agent JSON-parse failure → `synthesize()` → `FALLBACK`, route mid-stream error, client reconnect guard, cookie tamper fail-open-to-reauth) — all real, all small, all matched to a specific failure shape. That much *is* L2 territory in isolation. But the load-bearing absence: **no circuit breaker** (`lib/mcp/client.ts:121–132`, called out in `study-system-design/audit.md` finding #2 and `study-ai-engineering/06-production-serving/05-retry-circuit-breaker.md`). During a 5-minute Bloomreach outage, every concurrent request burns 3 retries × ~12s = ~36s before failing — fine at 1 user, costly at 10+. Also missing: per-tool timeout, jitter on backoff, Anthropic-side throw handler (`study-testing/audit.md`: "the Anthropic throw path is not tested"), no fail-fast during cooldown. The pattern is recognized in the audits but the code is unchanged.

**TRADEOFF EVIDENCE — present, strong.** This is the surprise in the audit. `.aipe/rehearse-design-doc/` has 3 RFCs that *do* clear all four "warrants a doc" tests: RFC-001 (no database — encrypted cookie), RFC-002 (NDJSON over fetch+ReadableStream, not SSE), RFC-003 (deterministic supervisor in code, not LLM supervisor). Each names the rejected alternatives, the load-bearing fact that made the call, and the cross-cutting impact. **This is real L2/L3 signal** — the kind of "I can defend this choice against the obvious alternative" artifact that reads as senior. The catch: it's a *post-hoc* artifact (the docs were written for the recon family, not at the time of decision), and the docs themselves admit decisions about *model choice*, *prompt structure*, and *eval strategy* didn't earn an RFC. The tradeoff evidence is concentrated in the systems/architecture layer; it's thin on the AI layer specifically.

### One post-mortem of record

`.aipe/study-debugging-observability/05-auth-secret-flake-postmortem.md` documents the AUTH_SECRET flake fix (commit `e83a8e0`, fix lives in `test/mcp/auth.test.ts:117–122`). Symptom (1-in-N flake) → diagnosis (parallel-worker `process.env` mutation leaking across worker process reuse) → fix (`vi.stubEnv` + `vi.unstubAllEnvs` in `afterEach`) → verification (169 tests, repeated runs, clean). **This is the only commit in the repo with a real symptom→cause→fix arc.** L2 signal lives here, in this one specific war story. Nothing comparable exists for the AI layer (no documented prompt regression, no documented synthesize-misfire, no documented MCP rate-limit outage post-mortem).

### The critical correctness bug — named but unfixed

`lib/state/insights.ts:4` holds a *global* `Map<id, Insight>` per Vercel instance. `putInsights()` at line 30 calls `insights.clear()` (line 36) — correct for one user, **wipes another user's data** when two land on the same warm instance at ~10 concurrent users. Cross-guide convergence: named in `study-system-design/audit.md` (Ceiling 1, Top-3 #1), `study-distributed-systems`, `study-runtime-systems`, `study-database-systems`. ~30 LOC fix (session-key the Map by the `bi_session` UUID that already exists via `getOrCreateSessionId`). Recognized but not shipped. This is the *single highest-leverage correctness fix in the codebase* and it remains open.
