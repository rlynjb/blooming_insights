## Lens 2 — Focus

### Focus visibility

- No project CSS sets focus styles. `app/globals.css:1-80` declares color tokens (`:root` block, lines 3-15), `@theme inline` tokens (lines 17-23), body styling (lines 25-29), the global `button, input, .card { border-radius: 4px }` rule (lines 31-35), and the `bi-fade-up`, `bi-progress`, and `bi-dots` animations (with `prefers-reduced-motion` overrides at line 42 and lines 78-80). There is no `:focus`, `:focus-visible`, or `outline` rule. The project relies on user-agent default focus rings except where a component overrides them.
- Tailwind v4 is imported at `app/globals.css:1` (`@import "tailwindcss"`).
- One explicit removal: `components/chat/QueryBox.tsx:66` sets `outline: 'none'` inline on the text input and adds no replacement focus indicator. The QueryBox is currently behind a feature flag (`SHOW_QUERY_BOX = false` at `app/page.tsx:14`), so the only route where it would render is the feed, where it stays hidden by default.

### Focus on route change

- Navigation is via `next/link` (`InsightCard` → `/investigate/:id`, CoverageGrid firing tile → same, BackLink → `/`, "see recommendations" → `/investigate/:id/recommend`, ProcessStepper rows with `href` set). No `useEffect` in the scanned files calls `.focus()` or moves focus on mount. Focus behaviour on route change is whatever Next.js' App Router default provides.

### Focus on disclosure open/close

- `ToolCallBlock` (`components/investigation/ToolCallBlock.tsx:37-86`) is a `<button aria-expanded={expanded}>` whose body conditionally renders below. Focus stays on the trigger button across toggle (the button is not removed from the DOM).
- `StreamingResponse`'s "show/hide reasoning" (`components/chat/StreamingResponse.tsx:249-263`) behaves the same: trigger button persists, body conditionally renders below.
- Hypothesis rows use native `<details>` / `<summary>` (`components/investigation/EvidencePanel.tsx:215-262`); browser-default focus handling applies. `summary` is styled with `listStyle: 'none'` (line 222) and `cursor` toggles between `'pointer'` and `'default'` depending on whether `h.reasoning?.trim()` is truthy (line 223). No disclosure marker is added back.

### Focus restoration

- Not implemented. The "× clear" button (`app/page.tsx:580-595`) unmounts the `StreamingResponse` article; no code restores focus to wherever the query was submitted from. The reconnect button (`app/page.tsx:663-687`) triggers `window.location.href = '/'` — a full navigation that resets focus to the document.
